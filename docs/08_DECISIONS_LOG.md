# Decisions Log

## 1. Telegram Mini App instead of plain Telegram bot

Решение: делать не обычного бота с кнопками, а Mini App.

Причина:

- Удобнее формы.
- Лучше UX для поиска.
- Можно делать карточки игроков/команд.
- Проще расширять интерфейс.

## 2. Main sections

Финальные разделы:

```text
Анкета
Найти команду
Найти игрока
Помощь
Настройки
```

## 3. Region replaced with city

Изначально было поле “Регион”, но его заменили на “Город”.

Причина:

- Комьюнити может искать людей из своего города.
- Удобно для локального общения.
- Можно использовать в фильтрах.

## 4. Profile is required before search

Решение: для поиска команды/игрока нужна заполненная анкета.

Причина:

- Без анкеты приглашения и заявки неинформативны.
- Меньше спама.
- Лучше качество matchmaking.

## 5. Player search uses profiles, not separate player entity

Игрок = `Profile`.

Нет отдельной таблицы `players`, потому что вся нужная информация уже в анкете.

## 6. Team has captain and members

Команда создается пользователем-капитаном. Автор команды автоматически добавляется в `team_members`.

## 7. Applications and invites are separate entities

`team_applications` — заявки в команду.

`player_invites` — приглашения между игроками.

Причина:

- У них разная логика прав.
- У них разные получатели.
- У них разные статусы и UI.

## 8. Help and Settings postponed

Помощь и Настройки будут проработаны позже.

## 9. Tone

Тон интерфейса: игровой, короткий, понятный, с легким юмором.

Не делать слишком корпоративно.
